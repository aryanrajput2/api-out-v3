## 📋 Quick Reference

### Workflow Steps
```
Test → Create Release → Approve → Switch to Admin → Deploy
```

### Key Environments
- **Test**: `http://localhost:8000` (Development)
- **Staging**: `https://staging-api.example.com` (Pre-production)
- **Admin**: `https://admin-api.example.com` (Production)

### Release Status
- **Pending Approval**: Waiting for approval
- **Approved**: Ready to deploy
- **Deployed**: Live in production

### Dashboard Access
1. Click "Dashboard" button (top right)
2. Click "Deployment" in sidebar
3. Manage environments and releases

## 🚀 Getting Started Now

### 1. Access Dashboard
```
Click "Dashboard" button (top right)
```

### 2. Go to Deployment
```
Click "Deployment" in sidebar
```

### 3. Create Release
```
Fill in version and changes → Click "Create Release"
```

### 4. Approve Release
```
Click "Approve Release" button
```

### 5. Switch Environment
```
Select "Admin" → Click "Switch"
```
